# 我的检查规则（L3 用户自定义）

这是**用户自定义规则文件**，初始为空。规则跟着 skill 走（不绑定具体报告目录），所有报告共用。

**如何添加规则**（不用手动编辑这个文件）：
直接在对话里告诉 Claude 你想额外检查什么，Claude 理解后会自动把规则追加到下方"## 我的规则"章节。例如：
- "我们事务所要求检查关联方披露是否完整，关联交易金额要与交易明细披露一致"
- "金融行业客户要重点检查贷款五级分类与拨备覆盖率披露"
- "本所内部规范要求核对实收资本与工商登记信息一致性"

Claude 会把你的需求整理成"自然语言描述 + 适用场景 + 期望结果"格式写入下方，下次检查报告时自动加载执行。

**可以描述什么**：
- **特定披露要求**：关联方披露、持续经营披露、或有事项披露、期后事项等
- **行业特殊科目**：金融业（贷款、拨备）、房地产业（存货-开发产品/开发成本）、制造业（生产成本/制造费用明细）等
- **事务所内部规范**：特定科目披露口径、附注编号规范、签字盖章要求等
- **客户专属要求**：母公司报表勾稽、分部报告勾稽、监管报送口径等
- **复核关注点**：上一年度审计调整事项是否入账、重大会计估计变更披露等

**规则写好后**：Claude 检查每份报告时都会读取并执行这些规则。规则失效或需要修改时，直接在对话里说"删掉关联方那条规则"或"修改某条规则的检查口径"即可。

## 示例规则（仅供参考，未启用）

以下示例展示规则格式，初始未启用（放在示例章节）。你可以参照格式在对话中描述自己的需求，Claude 会按此格式写入"## 我的规则"章节。

### 示例 1：关联方披露完整性

- **规则描述**：检查关联方披露是否完整，关联方交易金额应与"关联方及关联交易"附注中分项披露的金额合计一致；重大关联交易需单独披露定价政策。
- **适用场景**：所有报告，重点关注关联交易金额较大的客户。
- **期望结果**：若发现关联交易合计数与附注明细不一致，或缺失定价政策披露，记为"提示"（蓝），提示人工复核。

### 示例 2：金融行业贷款拨备

- **规则描述**：金融行业客户应披露贷款五级分类（正常/关注/次级/可疑/损失）金额，并计提相应贷款损失准备；拨备覆盖率（贷款损失准备/不良贷款）应与监管指标披露一致。
- **适用场景**：银行、小贷公司、消费金融等金融机构客户。
- **期望结果**：若五级分类金额合计 ≠ 贷款总额，或拨备覆盖率与监管指标披露不一致，记为"异常"（黄）。

### 示例 3：实收资本与工商登记核对

- **规则描述**：核对实收资本（股本）附注披露的股东出资信息与公司章程/验资报告一致性（若有），股东出资比例合计应为 100%。
- **适用场景**：新设公司、增资扩股、股权转让当年。
- **期望结果**：若股东出资比例合计 ≠ 100%，或存在出资未到位情况未充分披露，记为"异常"（黄）。
